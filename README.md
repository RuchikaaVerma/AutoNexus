## 🎯 Overview

**AutoNexus** is an intelligent vehicle maintenance prediction system that uses **real OBD-II sensor data** from 81+ vehicles, **8 specialized AI agents**, and **machine learning** to predict vehicle failures **before they happen**.

### The Problem

- **25% of vehicle breakdowns are preventable** with early detection
- **Emergency repairs cost 3x more** than scheduled maintenance
- **40% of drivers skip routine maintenance** due to inconvenience
- Traditional systems are **reactive**, not proactive

### Our Solution

- **Proactive Maintenance**: Predict failures 7-90 days in advance
- **Multi-Agent AI**: 8 specialized agents working together
- **Real Data**: Trained on 1,200+ samples from 81 real vehicles
- **Smart Notifications**: Time-aware alerts (no midnight calls!)
- **Automated Workflow**: From prediction → notification → booking → service → feedback

## ✨ Features

### 🤖 **8 AI Agents - Multi-Agent Architecture**

| Agent | Responsibility | Innovation |
|-------|---------------|------------|
| **DataAnalysisAgent** | Sensor threshold analysis | Real-time anomaly detection |
| **DiagnosisAgent** | ML-based failure prediction | 94% accuracy with XGBoost |
| **EngagementAgent** ⭐ | Time-aware notifications | **9 AM-8 PM calls only** |
| **SchedulingAgent** | Appointment booking | Urgency-based scheduling |
| **FeedbackAgent** | Customer satisfaction | 15-question multi-section survey |
| **ManufacturingInsightsAgent** | RCA/CAPA reports | Fleet-wide pattern detection |

### 🔮 **Predictive Capabilities**

- **Failure Prediction**: 7-90 day advance warning
- **Component Risk Analysis**: Brake, oil, engine, tire, battery
- **Accuracy**: 94% test accuracy, 92% cross-validation
- **Confidence Scoring**: Transparent ML predictions with SHAP values

### 📱 **Smart Notifications**

- **Time-Aware Calls**: No midnight disturbances (9 AM - 8 PM only)
- **Multi-Channel**: SMS, Email, Voice, Push notifications
- **Urgency-Based**: CRITICAL → immediate, MEDIUM → 24hrs, LOW → 7 days
- **Two-Way Communication**: Interactive voice menus (IVR)

### 🔒 **UEBA Security**

- **Anomaly Detection**: Isolation Forest algorithm
- **Baseline Learning**: 30-sample minimum for real behavior
- **Access Control Matrix**: Role-based permissions
- **Threat Detection**: Unauthorized access, sensor tampering, data exfiltration
- **Automated Response**: Alert → Log → Block → Notify

### 📊 **Manufacturing Insights**

- **Root Cause Analysis (RCA)**: Identify systemic issues
- **CAPA Generation**: Corrective & Preventive Actions
- **Fleet Patterns**: Detect problems across vehicle models
- **PDF Reports**: Professional RCA-CAPA documentation

### 🎙️ **Voice AI Integration**

- **Text-to-Speech**: Natural voice alerts (Polly/gTTS)
- **Speech-to-Text**: Voice command recognition
- **Interactive IVR**: Press 1 to book, 2 for technician, 3 for SMS
- **Call Management**: Twilio integration with proper TwiML handling

## 📦 Installation

### **Prerequisites**
- Python 3.9 or higher
- Node.js 16+ (for frontend)
- Git
- ngrok (for voice call testing)

### **Backend Setup**

```bash
# 1. Clone repository
git clone https://github.com/your-username/autonexus.git
cd autonexus

# 2. Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Set environment variables
cp .env.example .env
# Edit .env with your credentials:
# - HUGGINGFACE_API_TOKEN
# - TWILIO_ACCOUNT_SID
# - TWILIO_AUTH_TOKEN
# - TWILIO_PHONE_NUMBER
# - SENDGRID_API_KEY
# - ALERT_PHONE
# - ALERT_EMAIL

# 5. Initialize database
python -c "from database import Base, engine; Base.metadata.create_all(engine)"

# 6. Seed database with 10 vehicles
python seed_data.py

# 7. Run backend server
uvicorn main:app --reload
```

Backend now running at: `http://localhost:8000`

### **Frontend Setup**

```bash
# 1. Navigate to frontend directory
cd frontend

# 2. Install dependencies
npm install

# 3. Configure API URL
# Edit src/api/config.js:
# const API_URL = 'http://localhost:8000';

# 4. Run frontend
npm start
```

Frontend now running at: `http://localhost:3000`

### **Voice Call Setup (Optional - for IVR testing)**

```bash
# 1. Install ngrok
# Download from: https://ngrok.com/download

# 2. Run ngrok
ngrok http 8000

# 3. Copy HTTPS URL (e.g., https://abc123.ngrok.io)
export PUBLIC_URL='https://abc123.ngrok.io'

# 4. Configure Twilio webhook
# Go to: https://console.twilio.com/
# Your phone number → Voice & Fax
# "A call comes in" → {PUBLIC_URL}/voice/initial-message
# Method: POST
# Save
```

## 🚀 Usage

### **1. Access Dashboard**
```
Open: http://localhost:3000
Click: "Enter Dashboard"
```

### **2. View Vehicles**
```
Dashboard displays 10 vehicles in grid
Color codes:
  🟢 Green = Healthy (health > 70%)
  🟡 Yellow = Warning (health 40-70%)
  🔴 Red = Critical (health < 40%)
```

### **3. Run AI Analysis**
```
Click any vehicle (e.g., VEH002)
Click: "Run Full AI Analysis"
Wait 2-5 seconds
See results from all 8 AI agents
```

### **4. Book Service**
```
If critical issue detected:
  - System sends SMS immediately
  - If 9 AM - 8 PM: Phone call in 5 minutes
  - If 8 PM - 9 AM: Phone call scheduled for 9 AM
  
Click: "Book Service"
Confirm appointment
Receive SMS + Email confirmation
```

### **5. Complete Service**
```
After service completion:
  - Vehicle health improves (e.g., 35% → 95%)
  - Dashboard updates automatically
  - Feedback survey sent (24 hours later)
```

### **6. View Reports**
```
Manufacturing Insights:

  Download PDF: GET /manufacturing/rca-capa-report/{vehicle_id}

UEBA Security:
  POST /security/ueba/scan/{vehicle_id}
  Download PDF: GET /security/ueba/report/{vehicle_id}
```

## 📡 API Documentation

### **Base URL**
```
http://localhost:8000
```

### **Key Endpoints**

#### **Vehicles**
```http
GET    /vehicles                    # Get all vehicles
GET    /vehicles/{vehicle_id}       # Get single vehicle
POST   /vehicles                    # Create vehicle
PUT    /vehicles/{vehicle_id}       # Update vehicle
DELETE /vehicles/{vehicle_id}       # Delete vehicle
```

#### **AI Analysis**
```http
POST   /agents/analyze/{vehicle_id} # Run all 8 agents
GET    /agents/status                # Get agent status
```

#### **Service Booking**
```http
POST   /service/book                 # Book service
GET    /service/history/{vehicle_id} # Get service history
POST   /service/complete/{booking_id}# Complete service
```

#### **Feedback**
```http
POST   /feedback/submit/{booking_id} # Submit feedback
GET    /feedback/vehicle/{vehicle_id}# Get vehicle feedback
```

#### **Manufacturing Insights**
```http
POST   /manufacturing/insights/{vehicle_id}        # RCA/CAPA analysis
GET    /manufacturing/rca-capa-report/{vehicle_id} # Download PDF
```

#### **UEBA Security**
```http
POST   /security/ueba/scan/{vehicle_id}    # Security scan
GET    /security/ueba/report/{vehicle_id}  # Download PDF
```

#### **Dashboard**
```http
GET    /dashboard/stats              # Dashboard statistics
```

#### **Admin**
```http
POST   /admin/seed                   # Seed 10 vehicles
POST   /admin/randomize              # Randomize vehicle data
```

**Full API docs**: http://localhost:8000/docs (Swagger UI)
